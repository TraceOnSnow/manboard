## Why

manboard 已具备稳定的 Thread CRUD 与 JSON 存储，但当前前端只是按固定分区排列的粗粒度列表：记录灵感需要填写完整表单，所有条目使用相同状态交互，且界面缺少一个让人愿意每天打开的行动焦点。现在应在不扩大后端和数据模型范围的前提下，将它迭代成可实际试用的个人行动与记忆中枢。

## What Changes

- 重构 dashboard 的视觉系统与响应式应用外壳，采用明快、简约、低动效的工具型界面；桌面端提供分区导航和右侧详情抽屉，移动端保持单列与触手可及的快速记录。
- 新增全局快速记录：只输入标题即可保存为 Inbox 条目，之后可在详情中将其整理为待办、项目、研究或媒体记录。
- 新增“今日待办”的低摩擦工作流：今日碎事以勾选完成为主，而不要求维护完整生命周期状态。
- 调整长期内容的呈现：项目、研究和媒体按最近更新排序；已完成内容默认折叠；长期条目才显示明确的状态菜单。
- 为每个板块增加上下文新增入口，并以抽屉替代现有阻断式编辑弹窗来承载详情与编辑。

## Capabilities

### New Capabilities

- `dashboard-quick-capture`: 以 Inbox 为落点的全局快速记录与后续整理工作流。

### Modified Capabilities

- `dashboard-shell`: 将固定分区 CRUD 面板升级为响应式行动 dashboard，并修改分区排序、上下文新增、条目详情与状态交互要求。

## Impact

- **前端**：主要修改 `dashboard/src/App.tsx`、现有卡片/表单/API 类型，并新增 dashboard 布局、快速记录、分区和详情抽屉组件及设计 token。
- **后端/API**：不改变 FastAPI 路由、Thread 枚举或 JSON storage；Inbox 使用已有的 `type=other` 表示。
- **依赖**：预计新增一个一致的 SVG 图标库；不引入图表、路由、通知或外部服务依赖。
- **行为**：既有 Thread 数据不迁移、不丢失；现有未展示的 `other` 条目会在 Inbox 中可见。
